## Design Philosophy

### Core Principles

| Principle | Implementation |
|-----------|----------------|
| **Knowledge-First** | UI reflects the knowledge graph, not raw files. Every view answers "what does this code mean?" |
| **Unified Mental Model** | Single coherent system with cross-linked navigation. No siloed pages. |
| **Actionable Everywhere** | Every visible concept is inspectable, queryable, and operable |
| **Backend Parity** | 100% of MCP/REST operations exposed. No hidden capabilities. |
| **Human-Centric** | Optimized for human comprehension. Agents use MCP only. |

### Graph Types

#### 1. Call Graph
```
Shows function call relationships.

Node = Function
Edge = Calls (directed)
Color = Feature context
Size = Complexity / Call frequency
Opacity = Confidence level
```

#### 2. Import Graph
```
Shows file/module dependencies.

Node = File
Edge = Imports (directed)
Clusters = By directory
Color = Language
Thickness = Import count
```

#### 3. Inheritance Graph
```
Shows class/interface hierarchies.

Node = Class/Interface
Edge = Extends/Implements (directed)
Layout = Top-down tree
Color = Domain/Infrastructure
```

#### 4. Knowledge Graph
```
Shows semantic relationships.

Node = Entity (any type)
Edge = Semantic relationship
  - "part_of" (method â†’ class)
  - "uses" (function â†’ type)
  - "same_feature" (feature clustering)
Color = Feature context
Clusters = By feature area
```

### Search Modes

#### 1. Natural Language Search
```
Examples:
- "authentication functions"
- "most complex classes"
- "functions that call the database"
- "code related to payment processing"
- "unused functions"
```

#### 2. Structural Search
```
Query patterns:
- calls:hashPassword          (functions calling hashPassword)
- calledBy:loginHandler       (functions called by loginHandler)
- imports:@stripe/stripe-js   (files importing Stripe)
- extends:BaseController      (classes extending BaseController)
- implements:IAuthService     (classes implementing interface)
- inFile:src/auth/*           (entities in auth directory)
- complexity:>10              (high complexity functions)
```

#### 3. Semantic Search
```
Query patterns:
- feature:authentication      (entities in auth feature)
- purpose:"validate*"         (purpose contains validate)
- businessValue:"revenue*"    (business value mentions revenue)
- tag:security                (tagged with security)
- domain:billing              (classified as billing domain)
- infrastructure:database     (classified as database infra)
```

#### 4. Diagnostic Search
```
Query patterns:
- confidence:<0.5             (low confidence entities)
- clarification:pending       (needs clarification)
- orphan:true                 (no callers/callees)
- dead:true                   (unreachable code)
- unjustified:true            (missing justification)
- unclassified:true           (missing classification)
- stale:>7d                   (not re-indexed in 7 days)
```

### Real-time Updates

```typescript
// WebSocket connection for live updates
const ws = new WebSocket('ws://localhost:3100/ws');

ws.onmessage = (event) => {
  const { type, payload } = JSON.parse(event.data);

  switch (type) {
    case 'indexing:progress':
      updateIndexingProgress(payload);
      break;
    case 'justification:progress':
      updateJustificationProgress(payload);
      break;
    case 'ledger:entry':
      appendLedgerEntry(payload);
      break;
    case 'entity:updated':
      invalidateEntityCache(payload.entityId);
      break;
  }
};
```
